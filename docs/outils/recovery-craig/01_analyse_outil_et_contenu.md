# Analyse exhaustive de l’outil et du contenu source

## 1. Nature de l’outil à numériser

Les deux guides fournis ne sont pas de simples livres linéaires. Ils constituent un **outil de travail personnel et collectif** fondé sur :

- des textes courts d’introduction ou d’expérience vécue ;
- des thèmes de rétablissement ;
- des feuilles de travail ;
- des questions ouvertes ;
- des espaces de notes ;
- des invitations à l’action ;
- un langage de validation, d’encouragement, d’espoir et de responsabilité personnelle ;
- une orientation pair-aidance, trauma, autonomie et dignité.

L’outil web doit donc être conçu comme un **moteur de parcours réflexifs**, pas seulement comme une liseuse numérique.

## 3. Typologie des objets pédagogiques

### 3.1 Objet `Guide`

Un guide est un ensemble ordonné de modules. Il possède :

- titre ;
- sous-titre ;
- auteur ;
- édition ;
- langue ;
- notice de droits ;
- description courte ;
- audience cible ;
- avertissements de contenu ;
- statut de licence ;
- version éditoriale.

### 3.2 Objet `Module`

Un module correspond à un thème. Il peut contenir :

- titre ;
- slug ;
- résumé paraphrasé ;
- durée estimée ;
- niveau d’intensité émotionnelle ;
- tags ;
- texte source ou adaptation sous licence ;
- avertissement spécifique ;
- questions ;
- champs de notes ;
- actions recommandées ;
- ressources liées ;
- modules connexes.

### 3.3 Objet `Worksheet`

Une feuille de travail est un formulaire réflexif. Elle peut contenir :

- une consigne ;
- une à plusieurs questions ;
- champs texte long ;
- champs liste ;
- champs objectifs ;
- champs “personne de confiance” ;
- champs “moyens d’adaptation” ;
- sauvegarde automatique ;
- brouillon ;
- export ;
- reprise.

### 3.4 Objet `JournalEntry`

Entrée libre non forcément rattachée à un module. Champs recommandés :

- titre optionnel ;
- contenu ;
- humeur ressentie optionnelle ;
- tags personnels ;
- visibilité ;
- chiffrement ;
- date de création ;
- date de modification.

### 3.5 Objet `RecoveryPlan`

Plan personnel évolutif. Sections recommandées :

- signes que je vais moins bien ;
- choses qui m’aident ;
- choses à éviter ;
- personnes à contacter ;
- lieux sûrs ;
- phrases qui m’aident ;
- mini-actions réalisables ;
- ressources d’urgence ;
- préférences de partage.

### 4.2 Niveau d’intensité émotionnelle

| Niveau | Libellé | Usage |
|---:|---|---|
| 0 | Doux | Gratitude, petites choses, plaisir. |
| 1 | Réflexif | Objectifs, gestion du temps, conscience de soi. |
| 2 | Chargé | Injustice, frustration, colère, attentes. |
| 3 | Très sensible | Trauma, pardon, humiliation, pensées extrêmes. |
| 4 | Crise potentielle | Détresse aiguë, auto-agression, risque suicidaire. |

Règles :

- Les modules niveau 3 doivent afficher un avertissement préalable et proposer un bouton “choisir un module plus doux”.
- Les modules niveau 4 ne doivent jamais être un simple exercice : ils doivent afficher immédiatement des ressources d’aide et une invitation à contacter une personne réelle.

### 4.3 Type de réponse attendue

- texte libre ;
- liste de 2 ou 3 éléments ;
- plan en étapes ;
- contact de soutien ;
- réflexion sur expérience passée ;
- choix d’un moyen d’adaptation ;
- objectif personnel ;
- gratitude ;
- message à soi-même ;
- export/partage.

## 6. Principes d’adaptation éditoriale

### 6.1 Ne pas transformer en contenu médical

Les guides utilisent une parole expérientielle. L’adaptation web doit conserver cette nature :

- proposer des questions ouvertes ;
- éviter les échelles cliniques sauf validées et séparées ;
- ne pas interpréter les réponses comme symptômes ;
- ne pas dire “vous êtes dépressif” ou “vous êtes à haut risque” ;
- privilégier “voici des ressources” et “voici vos propres notes”.

### 6.2 Préserver le ton humain

Le contenu source repose sur :

- validation ;
- dignité ;
- responsabilité ;
- humour parfois punk ;
- chaleur ;
- langage direct ;
- espoir.

L’interface doit éviter :

- froideur clinique ;
- injonctions de performance ;
- gamification culpabilisante ;
- scores ;
- alertes agressives ;
- notifications intrusives.

### 6.3 Gérer les contenus sensibles

Chaque module doit avoir une fiche éditoriale :

```yaml
safety:
  intensity: 3
  topics:
    - trauma
    - abuse
    - shame
  preface_notice: true
  soft_alternative_module: "les-petites-choses"
  crisis_resources_visible: true
  allow_skip_questions: true
  allow_private_mode: true
```

### 6.4 Ne pas reproduire sans droits

Structure autorisée sans reproduction substantielle :

- titres ;
- résumé original rédigé par l’équipe ;
- description de l’objectif ;
- catégories ;
- exercices nouveaux inspirés par une méthode générale, sans copier le texte protégé ;
- champ permettant à un administrateur autorisé d’importer le contenu sous licence.

Structure nécessitant autorisation :

- texte intégral ;
- questions originales ;
- extraits longs ;
- PDF/EPUB généré depuis le livre ;
- lecture audio ;
- adaptation commerciale.

## 7. Modèle d’import de contenu

Le contenu doit être versionné, auditable et séparé du code.

```yaml
content_pack:
  id: craig-lewis-fr
  title: "Guides de rétablissement — Craig Lewis"
  language: fr
  rights_status: permission_required
  license_document_id: null
  guides:
    - id: un-jour-nouveau-fr
      source_file: survivre_a_limpossible.md
      edition: "3e édition"
      modules_count: 38
    - id: survivre-impossible-fr
      source_file: betterdays.md
      edition: "1re édition"
      modules_count: 26
```

## 8. Qualité éditoriale et nettoyage linguistique

Les fichiers sources contiennent des irrégularités typographiques et de traduction : apostrophes échappées, accents manquants, espaces, retours de ligne, éventuels mots mal reconnus. Avant intégration publique :

1. convertir les fichiers en UTF-8 propre ;
2. normaliser les apostrophes ;
3. corriger les titres ;
4. séparer texte, notes et feuilles de travail ;
5. créer une version de référence ;
6. faire relire par un francophone natif ;
7. faire valider par le titulaire des droits ;
8. enregistrer l’historique des corrections.

## 9. Règles de modération éditoriale

Pour tout contenu ajouté par l’équipe :

- pas de promesse de guérison ;
- pas de culpabilisation ;
- pas de langage stigmatisant ;
- pas de comparaison entre utilisateurs ;
- pas d’injonction religieuse ou politique ;
- avertissement sur les thèmes sensibles ;
- lien vers ressources humaines en crise ;
- possibilité de passer une question ;
- rédaction inclusive mais sobre ;
- phrases courtes ;
- niveau de lecture clair.

## 10. Indicateurs produit non médicaux

Mesures autorisées :

- nombre de modules commencés ;
- nombre de brouillons sauvegardés ;
- nombre d’exports ;
- nombre de moyens d’adaptation ajoutés ;
- taux d’abandon d’un écran ;
- utilisation du mode pause ;
- activation/désactivation des rappels.

Mesures à éviter ou encadrer :

- score de risque personnel ;
- inférence de diagnostic ;
- classement de vulnérabilité ;
- analyse émotionnelle automatique sans consentement ;
- surveillance par un facilitateur non consentie.
